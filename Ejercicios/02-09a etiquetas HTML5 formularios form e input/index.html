Formularios

Es para enviar informacion a un servicio en la red para que la procese. Tipo, es un componente web el cual tu rellenas con la informacion solicitada y luego este se enviara a un servidor.

GET/POST

El metodo get y post, son metodos de paso de datos. Estos se usan para enviar datos de un formulario al servidor. Ahora, la diferencia entre get y post es la siguiente.

GET
  • Es un comando del protocolo http, se realiza una peticion a un URL.
  • Los datos enviados se ven en la URL, siendo que esto puede ser un peligro de seguridad del servidor, ya que te permite editar los datos sin necesidad de completar el formulario, solo modificando la url
  • En la url, segun el tipo de dato, puede verse de manera rara. Por ejemplo; los caracteres no admitidos por el metodo ASCII, se ven entre %% y un comando.
POST
  • El paso por POST funciona exactamente igual, se siguen pasando los pares nombre, valor. Pero los datos se envían dentro del paquete de datos y no por la URL.
  • Parece más seguro el paso de parámetros por POST, pero lo cierto es que los datos que viajan mediante el método POST, lo hacen en la cabecera del paquete y esta cabecera es visible por cualquier persona que esté “escuchando” en la misma red que el cliente que está enviando datos.
